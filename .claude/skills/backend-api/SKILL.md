## When to use this skill:

- When creating or modifying API endpoints and routes
- When working on files in `/api/`, `/routes/`, `/controllers/`, or `/app/api/` directories
- When implementing RESTful resource patterns (GET, POST, PUT, PATCH, DELETE)
- When adding query parameters for filtering, sorting, or pagination
- When defining URL structures and naming conventions for endpoints
- When implementing HTTP status codes and response formats
- When adding API versioning
- When working with rate limiting or API throttling
