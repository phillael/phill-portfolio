## When to use this skill:

- When creating new React, Vue, Svelte, or other UI components
- When working on files in `/components/`, `/ui/`, or `/src/components/` directories
- When defining component props and TypeScript interfaces
- When implementing component state with useState, useReducer, or similar
- When deciding between composition vs prop-drilling patterns
- When refactoring large components into smaller, focused ones
- When creating shared/reusable component libraries
- When implementing render props or compound component patterns
- When adding default prop values
- When documenting component APIs and usage examples
