---

## When to use this skill:

- When writing any code in any language (.ts, .tsx, .js, .py, .rb, etc.)
- When naming variables, functions, classes, or files
- When choosing between naming conventions (camelCase, snake_case, PascalCase)
- When refactoring code for better readability
- When removing unused code, imports, or commented-out blocks
- When extracting common logic into reusable functions (DRY)
- When reviewing code formatting and indentation
- When ensuring consistent code style across files
- When deciding function size and single-responsibility
- When writing self-documenting code with meaningful names
