## When to use this skill:

- When deciding whether to add a code comment
- When writing comments for complex or non-obvious logic
- When adding JSDoc, docstrings, or type annotations
- When documenting function parameters and return values
- When explaining business rules or domain-specific logic
- When reviewing and cleaning up existing comments
- When removing outdated or redundant comments
- When choosing between a comment and refactoring for clarity
- When writing README sections or inline documentation
- When ensuring comments are evergreen and future-relevant
